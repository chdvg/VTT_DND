Contributing Guidelines
Thank you for your interest in contributing! We welcome improvements, bug fixes, new ideas, and constructive discussions. To keep the project healthy and easy to work with, please follow the guidelines below.

1. Before You Start
- Check existing issues and pull requests to avoid duplicates
- Open a new issue if you want to propose a feature or report a bug
- Keep discussions respectful and focused on the project

2. How to Contribute
Bug Reports
Include:
- A clear description of the problem
- Steps to reproduce
- Expected vs. actual behavior
- Logs, screenshots, or error messages if helpful
Feature Requests
Explain:
- What problem the feature solves
- Why it benefits the project
- Any ideas for implementation (optional)
Pull Requests
- Fork the repository
- Create a new branch for your change
- Make your updates with clear, descriptive commits
- Ensure your code builds and passes any tests
- Submit a pull request with a short explanation of what and why

3. Code Style
- Follow the existing style and structure of the project
- Keep changes focused and avoid unrelated edits
- Add comments where clarity is needed
- Write code that is easy to read and maintain

4. Communication
- Be respectful and constructive
- Assume good intentions
- Keep discussions on-topic and solution‑oriented

5. Licensing
By contributing, you agree that your contributions will be licensed under the same license as the project.

6. Code of Conduct
All contributors must follow the project’s Code of Conduct to ensure a welcoming environment.
